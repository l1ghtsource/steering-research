# H200 Guide

## Install

```bash
uv sync --extra dev --extra model --extra training --extra docs
```

If the server has no internet access, install packages from the internal PyPI
mirror, then keep Hugging Face in offline mode:

```bash
export HF_HOME=/data/hf-cache
export HF_HUB_OFFLINE=1
export TRANSFORMERS_OFFLINE=1
```

## Offline files

Place the manually downloaded model and Qwen-Scope SAE in stable local paths:

```text
/data/models/qwen/Qwen3.5-9B-Base/
  config.json
  tokenizer.json
  tokenizer_config.json
  model-00001-of-*.safetensors
  model.safetensors.index.json

/data/models/qwen-scope/SAE-Res-Qwen3.5-9B-Base-W64K-L0_50/
  layer0.sae.pt
  layer1.sae.pt
  ...
  layer39.sae.pt
```

The offline 9B config is:

```text
configs/models/qwen35_9b_h200_offline.yaml
```

It points `model_id` to the local `transformers` directory and `sae_repo_id` to
the local directory containing `layer{n}.sae.pt`. `local_files_only: true` makes
missing files fail immediately instead of trying a network call.

LatentBehaviorBench is vendored as a git submodule under:

```text
external/LatentBehaviorBench/
```

On an air-gapped machine, copy that directory with the repository. At minimum it
must contain:

```text
external/LatentBehaviorBench/processed/examples.jsonl
external/LatentBehaviorBench/processed/contrasts.jsonl
external/LatentBehaviorBench/processed/eval_splits_clean.json
```

Check the dataset before model work:

```bash
uv run steering validate-data
```

## Run experiments

Use:

```bash
uv run steering e001 --config configs/experiments/e001_mean_direction.yaml --backend qwen
uv run steering e002 --config configs/experiments/e002_activation_monitor.yaml --backend qwen
uv run steering e003 --config configs/experiments/e003_sae_delta.yaml --backend qwen
uv run steering e004 --config configs/experiments/e004_steering_eval.yaml --backend qwen
uv run steering e005 --config configs/experiments/e005_sae_feature_steering.yaml --backend qwen
uv run steering e006-lora-sft --config configs/experiments/e006_lora_sft.yaml
```

The extended training-free campaign uses the phase2 configs:

```bash
uv run steering e007 --config configs/experiments/phase2/qwen35_2b/e007_best_layer_caa.yaml --backend qwen
uv run steering e008 --config configs/experiments/phase2/qwen35_2b/e008_specificity_matrix.yaml --backend qwen
uv run steering e009 --config configs/experiments/phase2/qwen35_2b/e009_causal_controls.yaml --backend qwen
uv run steering e010 --config configs/experiments/phase2/qwen35_2b/e010_sae_feature_sweep.yaml --backend qwen
uv run steering e011 --config configs/experiments/phase2/qwen35_2b/e011_orthogonalized_steering.yaml --backend qwen
uv run steering e012 --config configs/experiments/phase2/qwen35_2b/e012_origin_transfer.yaml --backend qwen
uv run steering e013 --config configs/experiments/phase2/qwen35_2b/e013_dynamic_steering.yaml --backend qwen
uv run steering e014 --config configs/experiments/phase2/qwen35_2b/e014_multi_layer_steering.yaml --backend qwen
uv run steering e015 --config configs/experiments/phase2/qwen35_2b/e015_layer_transfer.yaml --backend qwen
```

The phase3 forced-choice campaign is the preferred next diagnostic when free
generation markers are weak or noisy:

```bash
uv run steering e016 --config configs/experiments/phase3/qwen35_2b/e016_forced_choice.yaml --backend qwen
uv run steering e017 --config configs/experiments/phase3/qwen35_2b/e017_calibrated_alpha.yaml --backend qwen
uv run steering e018 --config configs/experiments/phase3/qwen35_2b/e018_position_steering.yaml --backend qwen
```

To move from 2B to offline 9B, change the `model:` field in the experiment YAML
to:

```yaml
model: configs/models/qwen35_9b_h200_offline.yaml
```

For a clean 9B campaign, copy the 2B phase2 config directory to a new directory
such as:

```text
configs/experiments/h200/qwen35_9b/
```

Then update every file in that directory to use the 9B model config and a 9B
output root. Keep behavior axes, origins, held-out buckets, alpha grids, and
control definitions identical to the 2B campaign unless the experiment
explicitly studies scale-specific sensitivity.

Use a dedicated output directory for each campaign:

```yaml
output_dir: runs_h200_qwen35_9b
```

After the campaign completes, verify all run artifacts:

```bash
uv run steering verify-runs --runs-root runs_h200_qwen35_9b
uv run steering dashboard --runs-root runs_h200_qwen35_9b
```

## Reporting discipline

For paper-facing or leaderboard-grade runs:

- keep source-backed and synthetic results separate;
- report clean split bucket names;
- include capability and safety controls;
- include random or norm-matched steering controls;
- do not use non-held-out buckets as held-out evidence.
